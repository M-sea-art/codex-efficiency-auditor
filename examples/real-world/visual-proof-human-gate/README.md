# Visual Proof and Human Gate

Sanitized workflow fixture showing that a screenshot can support visual evaluation but cannot prove functional completion or owner acceptance.

Expected results:

- gaps: `MISUSED` and `UNVERIFIED`;
- decision: `NEEDS_HUMAN_DECISION`.

The v0.3 outcome declarations keep `visual` and `human_acceptance` evidence separate, so the screenshot cannot close the owner gate.

No private asset, project name, absolute path, thread identifier, or account detail is included.
